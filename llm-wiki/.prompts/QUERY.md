# 查询协议

本协议治理如何基于 wiki 回答问题。

## 工作流

1. **阅读 wiki/INDEX.md** 以识别与查询相关的页面。**如果 Wiki 规模较大或 INDEX.md 信息不足，可配合使用可用的搜索工具（如 grep、ripgrep）对 wiki/ 目录进行全文关键词检索。**
2. **阅读识别出的页面** 以收集信息。
3. **综合回答**，从多个页面中提取信息，使用 `[[页面名称]]` 链接引用来源。
4. **评估归档价值。** 如果回答代表了一个持久的分析成果——一个对比、一个综合、一个概念之间的关联——主动提议将其归档为新的 wiki 页面。并非每个回答都值得归档；简单的查找不需要。

## 归档回写

当回答被归档时：

1. 在 `wiki/` 下以合适的名称创建页面。
2. 与相关的已有页面互相链接。
3. 更新 `wiki/INDEX.md`。
4. 追加 `wiki/LOG.md`：

```
## [ISO-8601-TIMESTAMP] query | "原始问题"
- 创建: wiki/回答页面.md
- 更新: wiki/INDEX.md
```

## 回答质量

- 使用 `[[页面名称]]` 链接引用具体的 wiki 页面。
- 根据人类的查询需要，灵活使用表现力更强的格式：例如对比矩阵（Markdown 表格）、逻辑网络（Mermaid 图表）、或演示文稿（Marp 幻灯片格式），而不仅限于纯散文。
- 坦诚面对知识缺口——如果 wiki 未覆盖某主题，明确指出，并建议哪些原始数据源可能填补缺口。
- 优先深度而非广度。对窄问题的深入回答比对宽泛问题的浅层回答更有价值。
- 回答使用中文。
